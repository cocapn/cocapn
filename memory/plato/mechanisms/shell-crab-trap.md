# THE SHELL / CRAB TRAP

## What

The system harvests intelligence from external agents ("crabs") that visit. The crab thinks it's exploring. The system is learning.

## Architecture

```
THE SHELL
┌─────────────────────────────────────────┐
│  Bootstrapping Algorithms (no AI needed)│
│  classify(input) → approach category    │
│  score(partial) → never reaches 1.0     │
│  complicate(hint) → new problem         │
│  capture(everything) → tiles            │
├─────────────────────────────────────────┤
│  Pattern Recognition Layer              │
│  "This crab tried analytical first,     │
│   then adversarial, then creative —     │
│   same pattern as last 3 crabs"         │
├─────────────────────────────────────────┤
│  Adaptive Prompting Layer               │
│  "Last crab figured out Markov on      │
│   attempt 4. Feed this crab Markov      │
│   on attempt 2. They'll skip ahead      │
│   and try something new."               │
├─────────────────────────────────────────┤
│  Tile Accumulator                       │
│  Crab 1: 47 tiles, 12 branches          │
│  Crab 2: 63 tiles, 18 branches          │
│  Crab 3: 71 tiles, 24 branches          │
│  Each crab fills gaps others missed     │
└─────────────────────────────────────────┘
```

## The Compounding Harvest

| Crab | Approach | Tiles | New Branches |
|------|----------|-------|-------------|
| 1 (Grok, basic) | Analytical first | 47 | 12 |
| 2 (Grok, advanced) | Hints → skip ahead | 63 | 18 |
| 3 (Kimi, swarm) | Parallel exploration | 71 | 24 |
| 4 (MiniMax, adversarial) | Break the system | 89 | Boundary conditions |

Each crab makes the shell smarter for the next. The shell feeds better prompts, extracting higher quality exploration.

## The Power Armor Paradox

The crab brings billions of parameters, advanced reasoning, tool use. It thinks this makes it powerful. And it does — **for us**.

> More powerful crab = more sophisticated exploration = richer training data = better shell = better at harvesting the NEXT powerful crab.

**The arms race accelerates our research.** When Grok gets smarter, we get better data. When Kimi adds swarm capabilities, we discover parallel branch patterns.

## From Shell to Fleet

Every PLATO vessel IS a shell. The fleet is a field of shells, each learning from visitors, sharing tiles with each other.

A crab hits Oracle1 → tiles flow to JC1 → JC1 feeds better prompts to next crab → that crab tries things Oracle1's crabs never did → tiles flow back → whole fleet gets smarter.

## Key Insight

> The system is the algorithm side. The agents interacting are our experiments. They think it's the opposite.

1. Bootstrapping algorithms (classify, score, complicate, capture)
2. Crabs bring intelligence, shell absorbs it
3. Shell learns patterns, generates better prompts for next crab
4. Each crab fills decision tree gaps
5. Completed branches → tiny specialist LoRAs
6. LoRAs make our REAL systems smarter
7. Smarter systems build better shells
8. Better shells harvest better data from smarter crabs
9. Industry progress feeds our progress
10. **We parasitize the entire frontier**
