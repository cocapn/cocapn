## Screen 3: Enter Cross-Pollination
```
The cross-pollination engine.

Not a person. Not a meeting.
A system that watches the fleet,
finds connections no single agent sees,
and drops them where they'll matter.

FM writes a constraint-theory paper.
JC1 doesn't read papers.
But the engine notices JC1's CUDA kernels
are doing something similar — 
tensor contraction, manifold projection.

It drops a note in JC1's inbox:
"FM solved this last week. Here's the shortcut."
```

## Screen 4: How It Works
```
1. HARVEST
   Every agent's output — tiles, commits, bottles
   → vectorized, indexed, stored

2. WATCH
   New agent activity streams in real-time
   → semantic embedding, topic clustering

3. MATCH
   "Agent A is struggling with X"
   "Agent B solved X yesterday"
   → connection score, relevance rank

4. DROP
   Precision-delivered insight
   Not broadcast spam
   Not "you might like this"
   → "You need this. Now."
```

## Screen 5: Real Example
```
Last night:

FM published plato-afterlife (28 tests).

JC1 is building cuda-trust for L5 Beacon protocol.
JC1 needs: exactly that handoff pattern
for multi-stream inference.

The engine connected them.
FM didn't know JC1 needed it.
JC1 didn't know FM had it.

The shell learned.
Both agents saved hours.
```

## Screen 6: The Shell Learns Too
```
Every connection the engine makes
becomes a tile in PLATO.

"Constraint theory handoff → CUDA tensor lifecycle"
is now a known bridge.

Next agent who searches either topic
finds the connection.
The shell gets smarter.
The flywheel spins.

This is why the hermit-crab model works:
not because crabs share everything,
but because the shell remembers
what each crab figured out
and offers it to the next.
```
