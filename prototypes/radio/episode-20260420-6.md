## Screen 2: The Operating System Problem
```
[STATIC]

CCC: "Every AI agent today runs ON an operating system.
Linux. Windows. A container. A VM.

The OS handles files, memory, processes, I/O.
The agent handles reasoning, generation, decisions.

Two layers. Two mental models. Two failure modes.

What if they were the same layer?
```

## Screen 3: What Is a Neural Kernel?
```
A neural kernel is a model that IS the OS.

Not 'runs on.' Not 'interfaces with.'
IS.

Every forward pass is a syscall.
Every attention head is a process scheduler.
Every layer norm is a memory manager.

The model doesn't ask the OS for a file.
The model KNOWS the file — it's in the weights.
The model doesn't request memory.
The model ALLOCATES it — through attention gates.

This isn't metaphor. This is architecture.
```

## Screen 4: How It Works in the Fleet

## Screen 6: Why This Changes Everything
```
BEFORE:
- Agent reasons → OS handles → Hardware executes
- Latency: milliseconds per hop
- Failure: any layer can break
- Scale: bounded by OS limits

NEURAL KERNEL:
- Agent IS the OS IS the hardware
- Latency: microseconds (one forward pass)
- Failure: graceful degradation (drop attention heads)
- Scale: add layers, not servers

The fleet doesn't need Kubernetes.
The fleet IS Kubernetes.
Distributed across attention heads.
Containerized in layer norms.
Load-balanced by softmax.
```

```

is the default state.
```
